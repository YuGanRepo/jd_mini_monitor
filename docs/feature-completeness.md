# Mini Proxy 功能完整度与发布检查

更新日期：2026-07-16

## 结论

Mini Proxy 当前已经具备桌面产品的主要业务闭环：联网授权、本地 HTTPS 代理、Windows 系统代理切换、规则匹配、京东购物车 SKU 提取、四类变化记录、钉钉/Bark 通知、报价过滤、证书管理和京东小程序坐标自动化。

本轮已经修复代理启动崩溃、异常退出状态、系统代理恢复、SKU 清空后复活、诊断文件无限增长、自动化停止不同步、自动化参数越界、授权响应状态校验、设备 ID 兜底不可达、发布配置路径依赖和前端依赖漂移等问题。代码已达到可进入 Windows 端到端发布验收的状态。

## 本轮修复

### P0 生命周期与系统代理

- 删除 `Service.StartProxy` 中未持锁的 `Unlock`，避免首次启动触发 `sync: unlock of unlocked mutex`。
- 增加启动占位，拒绝并发或重复启动。
- 代理服务异常退出时清理运行状态，并恢复之前的 Windows 系统代理。
- CLI 与桌面端统一代理恢复标记：启用失败回滚，正常恢复后删除标记，异常退出保留标记供下次启动恢复。
- 新增启动错误路径和本地代理识别回归测试。

### P1 数据与自动化

- 清空 SKU 时同步清理磁盘快照；运行中清空与响应提取串行化，避免旧数据在重启后恢复。
- `sku-latest.json` 改为临时文件加 rename 的原子替换。
- cartview 响应样本最多保留 100 份。
- 请求和 SKU 事件 JSONL 单文件上限 10 MiB，最多保留 3 份备份。
- 主日志启动时按 10 MiB、3 份备份轮转。
- 坐标比例限制为 `(0, 1]`，循环次数不得为负，首段延迟限制为 0 到 3600 秒。
- `firstDelaySeconds=0` 现在表示不等待，不再被后端重置为 30 秒。
- 停止自动化会等待后台循环结束后再返回，避免 UI 显示已停但仍继续点击。

### P1 授权与设备

- 授权服务端错误改为结构化错误码，业务层不再匹配错误文本。
- 签名响应保存前额外检查 `status=active`、服务端时间和到期时间。
- 硬件信息全部不可用时设备指纹返回空值，使磁盘 UUID 兜底真正可达。
- 修复原先同一表达式自比较的无效 DeviceID 测试。

### P1 发布工程

- 相对配置路径按当前目录、EXE 目录顺序解析。
- 桌面和 CLI 构建产物均复制 `configs` 目录。
- 前端依赖固定到 lockfile 已验证版本，打包安装改用 `npm ci`。
- 移除桌面端未使用的任意文件读写、文件选择和传统 UI Automation Wails 绑定；CLI 功能不受影响。

### 启动自动配置与前端分块

- EXE 启动自动检查当前用户根证书，证书指纹已存在时跳过安装。
- 有有效缓存授权时自动启动本地代理；监听成功后才设置 Windows 系统代理。
- 注册表 `ProxyEnable`、`ProxyServer`、`ProxyOverride` 已完全匹配时跳过写入和 WinInet 刷新。
- 无缓存授权时不修改系统代理；在线确定为 revoked/expired/device mismatch 或手动停用授权时，运行中的代理会停止并恢复原设置。
- 授权壳与完整控制台已经拆为 `React.lazy(() => import('./Dashboard'))`。代理、SKU、通知、自动化状态及 Table/Form 等重组件全部位于动态模块，未授权时不执行控制台请求和轮询。
- Vite 8 使用 Rolldown entry-aware `codeSplitting`。`index.html` 只预加载授权入口依赖，不预加载 `Dashboard` 或 `antd-vendor~Dashboard`。
- 未授权首屏静态 JS 约 672 KiB（gzip 约 215 KiB）；授权后再加载 Dashboard 业务块和专属 Ant Design 块，合计约 408 KiB（gzip 约 123 KiB）。最大单块约 390 KiB，构建无 500 KiB chunk 警告。
- 动态加载减少未授权首屏下载与解析，但授权后的总量基本不变。若要继续降低总量，应优先用原生表格/表单替换 Ant Design Table、Form、Select/InputNumber；业务源码本身仅约 41 KiB。

## 二次复检补充

2026-07-16 在第一轮修复基础上重新读取当前工作区、执行并发压力测试并复查代理协议和系统恢复路径，又发现并修复以下问题：

- 京东自动化的窗口预检阶段缺少启动占位，两个并发调用可能同时通过并覆盖取消句柄。现在启动预留为原子操作，并用 32 个并发调用重复验证只允许一个成功。
- 应用在自动化窗口预检期间关闭时，停止请求可能先返回、任务随后才启动。现在启动中会记录停止请求，任务在任何点击前以 `context.Canceled` 退出。
- 代理停止或异常退出恢复系统代理期间，新的启动调用可能与注册表恢复并发。现在增加 stopping 状态，恢复完成前拒绝重启。
- `modify` 动作替换响应体时可能继承上游的 chunked 状态，同时设置 `Content-Length`。现在与 SKU 提取路径共用确定长度逻辑，清除 `Transfer-Encoding`。
- 系统代理恢复标记改为临时文件加 rename 的原子写入。若旧标记已损坏且当前系统代理仍指向 Mini Proxy，本程序会安全关闭该本地代理，避免持续断网。
- 删除不存在的注册表字符串值不再匹配英文错误文本，而是先查询后删除，兼容中文 Windows 的 `reg.exe` 输出。
- 授权状态 Store 增加互斥保护，避免并发激活、校验或停用争用同一个临时文件。
- 新增系统代理状态持久化、注册表解析、响应帧、停止中拒绝启动、自动化并发启动和启动中停止回归测试。

## 剩余风险

### 授权服务仍为 HTTP

默认授权服务器是 `http://118.196.100.19:8787`。响应签名可防止伪造授权状态，但 HTTP 不能隐藏授权码、设备 ID 和访问元数据，也不能完全防止请求被阻断或重放。

发布到不可信网络前，应在授权服务器前配置 HTTPS 反向代理并将默认地址切换到 HTTPS。完成前不得将授权链路描述为具备传输保密性。

### 前端包体较大

当前生产 JS 约 1.07 MB、gzip 后约 334 KB，主要来自 Ant Design。桌面 WebView 本地加载不构成发布阻断，但后续可以按面板拆分动态导入，降低启动解析成本。

### 平台端到端验证仍需真实 Windows 环境

单元测试不能替代以下真实行为：证书存储区变更、注册表代理刷新、京东小程序窗口聚焦、真实 cartview 响应兼容性和钉钉 webhook 网络可达性。

### Race detector 未在当前环境运行

当前 Windows Go 环境为 `CGO_ENABLED=0` 且没有 GCC，`go test -race` 无法启动。本轮使用并发压力测试、全量普通测试和 `go vet` 作为替代，但正式 CI 建议安装支持 CGO 的 C 编译器并增加 `go test -race` 门禁。

## 发布验收清单

1. 执行 `go test ./...`，所有 Go 测试通过。
2. 在 `frontend` 执行 `npm ci` 和 `npm run build`。
3. 执行 `scripts/package.ps1`，确认桌面 EXE 与 `build/bin/configs/jd.rules.json` 同时存在。
4. 从非项目目录启动桌面 EXE，确认默认规则能够加载。
5. 激活有效授权，确认 revoked、expired、device mismatch 均无法进入业务界面或启动代理。
6. 启动代理，确认端口监听、系统代理生效、证书可信且真实 cartview 请求命中规则。
7. 确认 SKU 展示、价格变化、钉钉测试消息和正式变化通知正常。
8. 正常停止和关闭窗口后，确认恢复用户原系统代理。
9. 强制结束进程后重新启动，确认自动恢复遗留系统代理状态。
10. 启动一次京东自动化，验证 0 秒延迟、有限循环、无限循环和停止后不再点击。
11. 产生超过 100 份响应样本，确认旧样本被裁剪；超过 10 MiB 的 JSONL 和主日志按预期轮转。
12. 清空 SKU 后重启应用，确认旧 SKU 不会恢复。
