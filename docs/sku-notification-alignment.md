# SKU 变动与通知对齐说明

更新日期：2026-07-16

Mini Proxy 的 SKU 变动和通知语义已按 `D:\github\jd-chrome-plugin` 当前实现对齐，同时保留桌面端已有的钉钉加签和自定义模板能力。

## 已对齐

- 支持插件 payload：`resultData.cartInfo.vendors[].sorted[]`，仅解析 `itemType === 1`。
- 继续兼容 Mini Proxy 现有的 `cartInfo.flatSkus` 和 `cartInfo.vendorList[].items`。
- 每次解析用新购物车快照整体替换旧快照；移出购物车的 SKU 不再残留。
- 首次发现 SKU 只建立基线，不发送变化通知。
- 变化类别：价格、库存、优惠、赠品。
- 价格字段：原价、到手价、优惠额、页面价、显示价、价格描述、还原价。
- 库存字段：库存状态、剩余库存；剩余库存差值必须严格大于配置阈值才通知。
- 优惠字段：PLUS 折扣/描述、促销增删改、降价提示/幅度、补贴信息。
- 赠品字段：赠品增删改。
- 分类开关：`price/stock/promo/gift` 独立启停。
- 推送格式：text、markdown。
- 渠道：钉钉与 Bark 可同时启用并发发送。
- 每批最多 3 个 SKU；消息超过 18,000 个字符时截断。
- 消息末尾附加设备 ID 尾 8 位识别码。
- 商品链接、支付链接、APP 扫码链接独立开关。
- 折扣系数默认 `0.97`，支持 `*N`、`N瓶/支`、整箱/原箱/原件包装折算。
- 报价匹配复用授权服务器 `/api/quote/match` 和当前授权 key/deviceId，缓存 10 分钟。
- 差价过滤开启后，仅过滤“已匹配且差价不大于阈值”的商品；未映射或查询失败仍通知。
- 通知配置保存后立即替换运行中代理的 Notifier，不再要求重启代理。

## 桌面端额外能力

- 钉钉 HMAC-SHA256 加签。
- 自定义 Go 模板；模板作为商品摘要，分类变化、报价和链接仍会追加。
- 抓包响应、解析事件和最新快照本地持久化与轮转。

## 数据源限制

是否能检测某类变化取决于实际响应是否包含对应字段。插件结构提供促销、赠品和剩余库存时可完整检测；`flatSkus` 本身字段较少，只能检测该结构实际携带的价格、数量和库存状态。Mini Proxy 不会根据缺失字段推造变化。

## 不属于 SKU 通知对齐的功能

- Chrome 标签页定时刷新和 alarm 调度。
- 购物车繁忙、无效 JSON、空购物车的暂停/恢复告警状态机。
- 浏览器弹窗、独立窗口和主题切换。
- 授权/报价管理后台及 MySQL 数据维护。

这些能力与 Windows 坐标自动化、代理抓包和桌面生命周期属于不同运行模型，应单独规划。
